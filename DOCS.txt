ОСОБЛИВОСТІ

- Без Strapi - власна система авторизації
- Без хардкодів - все через змінні оточення
- Візуалізований Mediator - панель розробника з графом взаємодії
- Brutalism Design - сирий, чесний, функціональний дизайн
- Модульна архітектура - Monorepo з packages
- WebSocket - повідомлення в реальному часі
- Повна українська локалізація

BRUTALISM DESIGN SYSTEM

- Палітра: Монохром (чорний/білий) + акцент #00ff00
- Рамки: 3px, 6px (товсті)
- Тіні: 8px 8px 0 (brutal shadows)
- Шрифти: Courier New (monospace)

MEDIATOR PATTERN

Централізована система комунікації між компонентами з візуалізацією.

Event Bus:
- Підписка на події (on, once)
- Відправка подій (emit)
- Відписка (off)
- Async обробка
- Error handling

Metrics System:
- Підрахунок подій
- Середній час обробки
- Підрахунок помилок
- Статистика по компонентам

History:
- Зберігання останніх 1000 подій
- Фільтрація за типом/джерелом
- Timestamp для кожної події

Visualizer:
- Події - список в реальному часі
- Метрики - статистика роботи
- Граф - візуалізація компонентів
- Компоненти - список зареєстрованих

Відкрити Mediator Visualizer: Ctrl+Shift+M

API ENDPOINTS

Auth:
- POST /api/auth/register - реєстрація
- POST /api/auth/login - вхід
- GET /api/auth/me - профіль

Questions (публічні):
- GET /api/questions - список питань (params: sortBy, page, limit, tag)
- GET /api/questions/:id - одне питання
- GET /api/questions/tags/all - всі теги з статистикою

Questions (потребують авторизації):
- POST /api/questions - створити питання
- PUT /api/questions/:id - оновити питання (тільки автор)
- DELETE /api/questions/:id - видалити питання (автор або admin)

Answers (публічні):
- GET /api/answers?questionId=X - відповіді на питання
- GET /api/answers?authorId=X - відповіді користувача

Answers (потребують авторизації):
- POST /api/answers - створити відповідь
- PUT /api/answers/:id - оновити відповідь (тільки автор)
- DELETE /api/answers/:id - видалити відповідь (автор або admin)
- PUT /api/answers/:id/accept - прийняти відповідь (автор питання)

Votes (потребують авторизації):
- POST /api/votes - проголосувати (body: {entityType, entityId, voteType})

Users:
- GET /api/users - список користувачів (params: sortBy, page, limit)
- GET /api/users/:id - профіль користувача
- POST /api/users/:id/block - заблокувати (admin)
- POST /api/users/:id/unblock - розблокувати (admin)

Search:
- GET /api/search?q=text - пошук питань

Таблиці:

users
  ├─ id (PK)
  ├─ username (UNIQUE)
  ├─ email (UNIQUE)
  ├─ password (hashed bcrypt)
  ├─ reputation
  ├─ role (user|admin)
  ├─ blocked
  ├─ blocked_at
  └─ timestamps (created_at, updated_at)

questions
  ├─ id (PK)
  ├─ title
  ├─ body (FULLTEXT)
  ├─ tags (JSON array)
  ├─ author_id (FK → users)
  ├─ views
  ├─ document_id
  └─ timestamps (created_at, updated_at, published_at)

answers
  ├─ id (PK)
  ├─ body (FULLTEXT)
  ├─ question_id (FK → questions)
  ├─ author_id (FK → users)
  ├─ is_accepted
  └─ timestamps (created_at, updated_at)

votes
  ├─ id (PK)
  ├─ user_id (FK → users)
  ├─ entity_type (question|answer)
  ├─ entity_id
  ├─ vote_type (up|down)
  └─ created_at
  └─ UNIQUE(user_id, entity_type, entity_id)

Індекси:

Users:
- INDEX idx_username (username)
- INDEX idx_email (email)
- INDEX idx_reputation (reputation)

Questions:
- INDEX idx_author (author_id)

- FULLTEXT idx_search (title, body)

Answers:
- INDEX idx_question (question_id)
- INDEX idx_author (author_id)
- INDEX idx_accepted (is_accepted)

Votes:

JWT токени:
- Термін дії: 7 днів
- Алгоритм: HS256
- Зберігання: localStorage (frontend)
- Передача: Authorization: Bearer {token}

Паролі:
- Bcrypt хешування
- Salt rounds: 10
- Мінімальна довжина: 6 символів

SQL захист:
- Prepared statements (pool.execute)
- Параметризовані запити
- Валідація вхідних даних

XSS захист:
- DOMPurify для Markdown
- React автоматичне екранування

ОСОБЛИВОСТІ РЕАЛІЗАЦІЇ

Система голосування:
- Один користувач може голосувати тільки раз за сутність
- Можна змінити голос (з up на down і навпаки)
- Підрахунок голосів динамічний через COUNT в SQL запитах
- Перевірка авторизації на бекенді

Система тегів:
- Теги зберігаються як JSON масив
- Автоматичне підрахування кількості питань для кожного тегу
- Фільтрація питань за тегом
- Підтримка кирилиці

Система репутації:
- Автоматичне нарахування/зменшення при голосуванні
- Відображення в профілі користувача
- Впливає на можливості користувача (планується)

Права доступу:
- Публічний перегляд контенту
- Авторизовані користувачі можуть створювати контент
- Автори можуть редагувати/видаляти свій контент
- Адміни можуть видаляти будь-який контент
- Тільки автор питання може прийняти відповідь

WEBSOCKET

const ws = new WebSocket('ws://localhost:3338/ws')

wsClient.on('questions', (data) => {

})

- VITE_WS_URL правильний в .env
- Перевірити консоль браузера
